# BUGS

## BUG-01 - Тогл «Только срочные» не фильтрует список объявлений

**Приоритет:** P1

**Предусловия:**
- Открыта главная страница платформы модерации объявлений.
- В списке отображаются объявления без ограничений по признаку срочности.

**Шаги воспроизведения:**
1. Открыть главную страницу.
2. Включить тогл `Только срочные`.
3. Дождаться обновления списка объявлений.

**Ожидаемый результат:**
- В списке отображаются только объявления с бейджем `Срочно`.

**Фактический результат:**
- После включения тогла в выдаче остаются объявления без бейджа `Срочно`.

## BUG-02 - Фильтр «Диапазон цен» работает некорректно

**Приоритет:** P1

**Предусловия:**
- Открыта главная страница платформы модерации объявлений.
- В списке отображаются объявления.

**Шаги воспроизведения:**
1. Открыть главную страницу.
2. В поле `От` ввести `1000`.
3. В поле `До` ввести `50000`.
4. Дождаться обновления списка объявлений.

**Ожидаемый результат:**
- В списке отображаются только объявления с ценой не меньше `1000` и не больше `50000`.

**Фактический результат:**
- После применения фильтра в списке остаются объявления с ценой выше `50000`.

## BUG-03 - Кнопка запуска автообновления на странице статистики не включает таймер

**Приоритет:** P1

**Предусловия:**
- Открыта страница статистики.
- Автообновление переведено в выключенное состояние.

**Шаги воспроизведения:**
1. Открыть главную страницу.
2. Перейти на страницу `Статистика` через интерфейс.
3. Нажать кнопку остановки автообновления.
4. Убедиться, что отображается состояние `Автообновление выключено`.
5. Нажать кнопку запуска автообновления.

**Ожидаемый результат:**
- Автообновление переходит в активное состояние.
- Кнопка меняет состояние на `Отключить автообновление`.
- Снова отображается блок с таймером обновления.

**Фактический результат:**
- После нажатия кнопки запуска состояние не меняется.
- Кнопка остаётся в состоянии `Включить автообновление`.
- Автообновление не возобновляется.

## BUG-04 - Прямой переход по URL `/stats` приводит к ошибке 404

**Приоритет:** P2

**Предусловия:**
- Пользователь не находится на странице статистики.

**Шаги воспроизведения:**
1. Открыть в браузере прямой URL страницы статистики `/stats`.

**Ожидаемый результат:**
- Открывается страница статистики.

**Фактический результат:**
- Отображается страница `Page not found`.

**Примечание:**
- При переходе на страницу статистики через интерфейс приложение работает корректно.
- Проблема воспроизводится именно при прямом открытии URL.
