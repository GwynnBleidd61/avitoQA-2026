# TESTCASES

## Общая информация
Платформа модерации объявлений содержит:
- главную страницу со списком объявлений (`/`, `/list`);
- страницу карточки объявления (`/item/:id`);
- страницу статистики (`/stats`).

Тест-кейсы ниже составлены для сценариев из задания 2.2.  
Формулировки сделаны проверяемыми и однозначными: в шагах указаны конкретные действия, а все проверки вынесены в ожидаемый результат.

---

## TC-01 - Главная страница: фильтр «Диапазон цен»

**Предусловия:**
- Открыта главная страница платформы модерации объявлений.
- На странице отображается блок фильтров.
- В блоке фильтров доступны поля минимальной и максимальной цены.

**Шаги:**
1. В поле минимальной цены ввести значение `1000`.
2. В поле максимальной цены ввести значение `50000`.
3. Дождаться обновления списка объявлений.

**Ожидаемый результат:**
- В списке отображаются только объявления с ценой не меньше `1000` ₽ и не больше `50000` ₽.
- Если подходящих объявлений нет, отображается корректное пустое состояние без ошибок интерфейса.
- Значения `1000` и `50000` остаются в соответствующих полях фильтра после применения.

## TC-02 - Главная страница: сортировка «По цене» по возрастанию

**Предусловия:**
- Открыта главная страница платформы модерации объявлений.
- На странице отображается список объявлений.
- На странице доступен блок сортировки.

**Шаги:**
1. В выпадающем списке «Сортировать по» выбрать значение «Цена».
2. В выпадающем списке «Порядок» выбрать значение «По возрастанию».
3. Дождаться обновления списка объявлений.

**Ожидаемый результат:**
- Список объявлений отображается в порядке возрастания цены.
- Каждая следующая цена в списке больше или равна предыдущей.
- Выбранные значения «Цена» и «По возрастанию» отображаются в элементах управления сортировкой после применения.

## TC-05 - Страница статистики: кнопка «Обновить»

**Предусловия:**
- Открыта страница статистики.
- На странице отображается контейнер управления обновлением статистики.
- В контейнере доступна кнопка «Обновить».
- Перед выполнением проверки в системе создано новое изменение данных, влияющее на статистику.

**Шаги:**
1. Зафиксировать текущее значение изменяемого статистического показателя.
2. Выполнить действие, которое должно изменить статистику.
3. Нажать кнопку «Обновить».
4. Дождаться завершения обновления.

**Ожидаемый результат:**
- Статистика обновляется по нажатию кнопки.
- Значение времени/статуса последнего обновления изменяется относительно зафиксированного до нажатия.
- Ошибки интерфейса при ручном обновлении не возникают.

## TC-08 - Мобильная версия: переключение между тёмной и светлой темой

**Предусловия:**
- Открыта главная страница сайта в мобильном разрешении.
- На странице доступен переключатель темы.
- Текущая тема интерфейса отображается в UI или в атрибуте/классе темы.

**Шаги:**
1. Зафиксировать текущую тему интерфейса.
2. Нажать на переключатель темы один раз.
3. Зафиксировать новое состояние темы.
4. Нажать на переключатель темы второй раз.

**Ожидаемый результат:**
- После первого нажатия тема интерфейса меняется на противоположную исходной.
- После второго нажатия тема интерфейса возвращается в исходное состояние.
- Состояние переключателя темы соответствует текущей теме интерфейса.
- Ошибки интерфейса при переключении темы не возникают.
